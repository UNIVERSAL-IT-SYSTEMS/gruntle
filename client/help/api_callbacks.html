API reference: callbacks

thickishstring will try to call special subroutines in your room script when things happen. You can implement these to add behaviour.

  • RoomDescription

    Returns a markup object describing the current room. This is used to show the player the current room description. This must be present.

    Note that this subroutine returns the room description, rather than directly telling the player! This subroutine should have no side effects and should not actually change anything.

  • Actions

    Returns a list of Action objects describing the actions currently available to the player.

    This subroutine should have no side effects and should not actually change anything.

    If this is not present, no actions appear in the tab at the bottom of the screen (although they can appear as links in the room description).
