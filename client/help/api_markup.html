API reference: markup

ThickBasic markup represents text that is going to be shown to the player. Objects and expressions can be embedded in markup which are rendered specially. For example, player names are highlighted.

Markup look like strings, although delimited with double quotes: 

m1 = "Hello, world!"

Markup can span multiple lines. If there is a blank line, that is treated as a paragraph break. Otherwise, whitespace is all collapsed. 

m2 = "This is
  all on one
    line.

But this is another paragraph."

To embed an expression, use curly braces. The expression will be evaluated and the value converted to a markup object. 

count = 3
m3 = "The number of the counting is {count}."

Markup can be concatenated with +. It is also possible to embed markup inside markup. m4 and m5 in the following are equivalent: 

if male then greeting="Sir" else greeting="Madam"
m4 = "Good day, {greeting}."
m5 = "Good day, " + greeting + "."

Markup is immutable, and the contents cannot be read in any way. (Because sometimes the content will depend on the player's browser settings, and the server doesn't know those.)
